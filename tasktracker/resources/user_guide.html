Task Tracker — application guide

This guide ships with the app and is read-only. Use Help → User guide, tab My notes, for your own conventions (what goes in each field).

Tasks

  • New Task (title, received, due), Save Task (persist the detail form), Close Task (close + on-close recurrence if configured) — toolbar, Task actions row at the top of the detail pane, and shortcuts (defaults Ctrl+N, Ctrl+S, Ctrl+Shift+C; change under Settings).
  • Two-column detail layout — the core fields are split for readability: the left column (wider) holds Ticket, Title, a combined Status / Impact / Urgency / Priority row, and Description (which grows to fill available height); the right column holds Received, Due, Closed, Category, Sub-category, Area, For person, and Next milestone.
  • Inline sections vs. tabs — below the core fields, Todos, Blockers, and Recurring stay visible inline as stacked group boxes; Notes and Activity share a tab strip that fills the remaining vertical space, so long histories or rich notes no longer push everything else off-screen. You can reorder both groups under Settings → Customize task panel layout….
  • Blank dates — Due and Closed date boxes stay blank until you set them; use the Today button to fill with today's date or Clear to empty them again. A blank date saves as "no date" (it does not default to today).
  • Category / Sub-category / Area — choose vault-specific taxonomy values for each task (Area is the task-level assignment; category/sub-category reflect that hierarchy).
  • For person — optional attribution for who the task is for (first name, last name, employee ID).
  • Impact / Urgency / Priority — priority is computed (ServiceNow-style matrix). Use Priority matrix… on the toolbar to pick impact/urgency pairs. Changing I or U writes audit entries and a system note when priority changes.
  • Statuses — open, in progress, blocked, on hold, cancelled, closed.
  • Next milestone — derived from the first open todo (in list order) that has a milestone date.

Todos

  • Ordered list; Up / Down reorders. Completing todos refreshes next milestone.
  • Optional milestone date per todo. When adding or editing, tick Milestone date to set one, or untick to clear it.
  • Edit a todo — select it and click Edit…, or double-click the row. You can change the title and set / change / clear the milestone; an empty title is ignored (the original title is kept). The parent task's Next milestone updates automatically when the first open todo's milestone changes.

Notes

  • Rich text; New note / Save note. System notes (from priority changes) are read-only in the editor.
  • Versions are kept when you edit a user note.

Blockers

  • Separate blocker rows; clearing a blocker may move status off blocked when none remain open.

Recurring

  • Enable on a task; choose on close vs scheduled (scheduled generation is stored for later automation).
  • Template lines: Title|days_after_received per line for todos on the next instance.
  • Weekend/holiday skips use Holidays tab and rule checkboxes.

Calendar

  • Toggles for due, milestones, received, closed; option to include closed tasks.
  • Set current task due date to selected day applies the calendar selection to the task selected on the Tasks tab.
  • Legend colors; list sorted by priority within a day.

Editing a task from the Calendar tab

  • Double-click any event in the day list (or highlight it and click Edit selected task…) to open a focused quick-edit dialog. The dialog covers the fields you usually tweak from a calendar view: Status, Due, Closed, Impact / Urgency (with a live priority readout), For person, and Area. Flipping status to Closed from here spawns the next recurring instance just like closing from the Tasks tab.
  • Todos / milestones inside the dialog — the same quick-edit dialog also embeds the task's todo list so you don't have to switch back to the Tasks tab just to shuffle a milestone. Use Add…, Edit…, Mark done, or Delete; double-click a row to edit it. The Shift selected: N day(s) / Business days / Apply to selection strip above the table shifts only the highlighted rows by the given number of days (business days skips weekends and configured holidays).
  • Also shift todo milestones — tick the checkbox below the core fields and every attached todo milestone is shifted by the same number of days you just changed the task's Due date by, using business-day math when the Business days sub-toggle is on. Leave the box unticked to move only the task's Due date and leave milestones alone.
  • Notes snapshot — the lower half of the dialog shows the task's notes (most recent first) as a short read-only list with a snippet. Double-click a line to pop open a read-only viewer with the full content. Editing notes still lives on the Tasks-tab editor — the calendar dialog is a reschedule-oriented surface.
  • The title shown in the dialog is read-only — renames belong on the full editor. Use the dialog's Open in Tasks tab for full edit link (or the Calendar tab's Open in Tasks tab button) to jump back to the full editor whenever you need descriptions, blockers, or recurrence.
  • If the task you picked isn't visible on the Tasks tab after jumping (for example, your search filter or "Include closed" toggle hides it), you'll get a status-bar nudge to clear the filter.

Bulk date shifts (re-tasking multiple items at once)

When a priority change or outage causes a week or more's worth of rescheduling, the Edit menu has two bulk-shift tools that share a common preview + undo flow. Both are business-day aware by default (skipping weekends and any holidays configured on the Holidays tab).

  • Edit → Shift selected tasks… — on the Tasks tab, hold Ctrl or Shift to multi-select rows in the task list, then pick this menu (or right-click and choose Shift dates…). You'll be asked for the number of days to move the selection; tick Include todo milestones to fan the same delta out to every todo in the selected tasks.
  • Edit → Slip schedule from date… — shift every task with a Due date on or after an anchor date by N days at once. Filters let you narrow the scope by For person, Area, Minimum priority, or Status; closed and cancelled tasks are excluded by default.
  • In the calendar quick-edit dialog, the Shift selected strip shifts the currently-highlighted todo rows only — useful when you want to re-stagger a couple of milestones without moving the parent task's Due date.

Every shift shows a preview table first, listing each item that would change, its old date, and its new date. Rows that would land on a weekend, a configured holiday, or that resolve to a no-op (nothing to shift) are tinted so you can spot them before applying. Click Apply to commit or Cancel to back out.

After applying, Edit → Undo last bulk shift (also Ctrl+Shift+Z, or the same action on the task-list right-click menu) reverts the most recent bulk operation in one click. Undo only covers the last bulk shift; edits you make to those same tasks in between (for example, saving a new Due date from the full editor) are detected and skipped during the revert so they're not accidentally overwritten.

Reports & export

The Reports tab has a list of management-style reports on the left, parameter controls in the middle, a results table, and a small summary panel. Pick a report, adjust the parameters, click Run report. The last-used parameters for each report are remembered between sessions in ui_settings.json.

  • WIP & aging — open tasks bucketed by age in calendar days (0-7 / 8-30 / 31-90 / 90+) cross-tabbed by status and priority. Parameter: As of date.
  • Throughput — closed-task counts over time, grouped weekly (Mon-Sun) or monthly. Parameters: From, To, Period (weekly/monthly), and an optional Group by split (none, by category, by for-person). "For-person" matches the For person field on the task form — the person the task is for, not the person entering it.
  • Workload — per for-person open count, overdue count, P1 / P2 counts, oldest received date, and average age. Tasks with no assigned For person appear in a (unassigned) bucket. Parameter: As of date.
  • SLA performance — for tasks closed in the window, how often the due date was met, average days late, and a per-category breakdown. Closed tasks with no due date are reported separately as no_sla and excluded from the miss-rate math. Parameters: From, To.
  • Category mix — per category, received vs closed in the window plus current open count, with a net-change column to show which workstreams are growing vs being burned down. Parameters: From, To.
  • Weekly status — composite "what's closed in the last 7 days, what's open by status, what's due in the next 7 days, and what's currently blocked", sized for an emailable management readout. Parameter: As of date.

For each report you can:

  • Export this report (CSV)… — write the rows you just saw to a CSV file (column order matches the table).
  • Copy summary to clipboard — copy a short multi-line text block (3-10 lines) suitable for pasting into a Teams chat or status email.

The File menu offers four export pathways:

  • Export tasks to CSV… / Export tasks to Excel… — flat single-sheet dump of every task, useful as a raw-data feed.
  • Export rich workbook (xlsx)… — one Excel file with multiple sheets: Summary (KPI block), Tasks (with computed age_days, days_to_due, days_late, was_closed_late), Todos, Notes (HTML stripped to plain text), Activity (audit log), and one sheet per report (WIP_Aging, Throughput_Weekly, Throughput_Monthly, Workload, SLA, CategoryMix, WeeklyStatus). Every data sheet has a frozen header row and an Excel auto-filter so it is immediately ready for sorting and pivoting.
  • Export reports bundle (CSVs in folder)… — write one CSV per report (wip_aging.csv, throughput_weekly.csv, throughput_monthly.csv, workload.csv, sla.csv, category_mix.csv, weekly_status.csv) into a folder you choose. Use this when you want to email or upload a small set of focused files instead of one large workbook.

Holidays

  • Add/remove dates; used when recurrence skips holidays (and shown on calendar when applicable).

Data & security

  • Data folder: app_data next to the project (or TASKTRACKER_DATA). Contains auth.json, encrypted DB at rest (tasks.db.enc), and plaintext DB only while running.
  • Master password on launch; backup app_data safely.

Vault persistence and switching

  • Remembering the last vault: Task Tracker remembers the folder you opened last and re-opens it automatically on the next launch. No more picking the same folder every run.
  • Pinning a default vault: In the vault picker, tick Always open this vault on launch before hitting Open / Create to pin it. The app will go straight to that vault on every subsequent start (the login dialog still prompts for the password, of course).
  • Recent vaults list: The picker has a Recent vaults drop-down that pre-fills the Open / Create path when you pick one. The list holds the last eight vaults you've opened.
  • Changing your mind: Click Clear pinned default in the vault picker to remove the pinned default. From the login dialog, a Switch vault… button relaunches the app straight into the picker. From the main window, Settings → Switch vault… does the same. Launching with --pick-vault on the command line also forces the picker.
  • Moved or deleted folders: If the saved vault folder no longer exists, the app falls through to the picker and shows a brief status-bar notice so you know why.
  • Where this is stored: Launcher state (last opened, pinned default, recents) lives in launcher.json under your OS's standard config folder. It is deliberately kept outside any vault so the app can still launch if the vault folder is unreachable, and so multiple vaults don't race to rewrite each other's launcher state.

Activity

  • Activity on the task panel merges field audit lines and note timeline.

Notifications

  • Routine confirmations (Task saved, Todo updated, Recurrence settings saved, Due date set to …, export "Saved.", etc.) appear briefly in the status bar at the bottom of the main window (about 4 seconds; the Close-task message stays a bit longer so you can read the new child ticket number). They do not steal focus — you can keep typing or press Ctrl+S again right away.
  • Genuine errors, warnings, destructive-action confirmations, and multi-line summaries (for example the reference-data Import complete report) still use pop-up dialogs so you must acknowledge them.

View

  • Theme… — switch the color scheme at any time via View → Theme. Four curated options ship out of the box, all rendered via Qt's Fusion style so every widget (buttons, inputs, checkboxes, date pickers, scrollbars, tabs) picks up the palette uniformly:
    • Light — soft off-white chrome with near-white input fields and a cool muted-blue highlight. First-launch default; balances readability with reduced glare compared to pure white.
    • Dark — dark gray backgrounds with light text, tuned for low-light sessions.
    • Light gray (reading) — soft gray page background and darker ink text; reduces glare further than Light without the eye fatigue of a full dark theme.
    • Sepia (reading) — warm parchment / sienna palette that's easier on the eyes for long reviewing / reading sessions.
    The menu entries are radio buttons — picking one unchecks the others. Changes take effect immediately across every open window and dialog and are saved to ui_settings.json, so the next launch reopens in your chosen theme.

Settings

  • Customize task panel layout… — reorder the sections below the core task fields. The dialog has two independently-reorderable groups: Inline sections (Todos, Blockers, Recurring — always visible) and Tabs (Notes, Activity — share a tab strip). Saved per vault in ui_settings.json.
  • Keyboard shortcuts… — change shortcuts for New Task, Save Task, and Close Task.
  • Date format… — choose how dates are displayed throughout the app. Presets include ISO yyyy-MM-dd (default), US MM/dd/yyyy, EU dd/MM/yyyy, short (17 Apr 2026), US long (Apr 17, 2026), and long (17 April 2026). A Custom… option accepts any Qt pattern (tokens: yyyy/yy year, MM month number, MMM/MMMM month name, dd/d day, ddd/dddd weekday); the dialog shows a live preview against today. The format affects date pickers, the Reports table + summary, the Tasks list, status-bar confirmations, milestone and Holidays displays, and the bulk-shift preview. CSV and Excel exports always stay ISO yyyy-MM-dd regardless of this setting, so spreadsheets sort dates correctly.
  • Manage categories and people… — maintain vault-specific Category → Sub-category → Area values and people attribution entries.
  • Export categories and people… / Import categories and people… — save/load taxonomy + people as JSON for vault synchronization.
